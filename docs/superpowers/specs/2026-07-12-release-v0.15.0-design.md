# Crucible v0.15.0 Release Design

## Goal

Publish Crucible `v0.15.0` from the merged default-model change and update the locally installed
Copilot CLI plugin to the same version.

## Release Scope

The release contains the user-visible default pairing merged in PR #57:

- Builder: Claude Opus 4.8 with effort `max`
- Critic: GPT-5.5 with effort `xhigh`

No runtime, schema, CLI, run-log, or dispatch changes are added during the release cut.

## Repository Changes

Create the release on `release/v0.15.0`, based on the current `origin/main` commit that contains
PR #57. Make only the release bookkeeping changes:

1. keep `## [Unreleased]` at the top of `CHANGELOG.md`;
2. move its current notes into `## [0.15.0] - 2026-07-12`;
3. set `.claude-plugin/plugin.json` to version `0.15.0`; and
4. set `.claude-plugin/marketplace.json`'s plugin version to `0.15.0`.

Released changelog sections and dated historical documents remain unchanged.

## Validation and Publication

Before opening the release pull request, run:

- `python3 scripts/check.py`;
- `python3 -m unittest tests.test_version_consistency`; and
- `python3 scripts/changelog.py section 0.15.0`.

The rendered section must be non-empty and describe the default pairing. Push the release branch,
open a pull request, wait for required checks, and squash-merge it into `main`.

After fetching the merged commit, create lightweight tag `v0.15.0` on that exact `main` commit and
push the tag. The existing Release workflow verifies that the tag is on `main`, matches both
manifests, and has non-empty changelog notes before publishing the GitHub Release.

Do not tag an unmerged branch, move an existing tag, or publish a manual release that bypasses the
workflow. If validation, pull-request checks, or the release workflow fails, stop at that gate,
diagnose the failure, and continue only after the repository is valid.

## Local Repository and Plugin Update

After the GitHub Release succeeds:

1. fast-forward local `main` to `origin/main`;
2. remove the merged default-model and release worktrees and their local branches;
3. prune stale worktree and remote-tracking metadata;
4. refresh the locally registered `crucible-marketplace`; and
5. run `copilot plugin update crucible@crucible-marketplace`.

Verify `copilot plugin list` reports `crucible@crucible-marketplace (v0.15.0)`. The local marketplace
continues to point at `/Users/sri/personal/crucible`, so the plugin update must happen after local
`main` is synchronized to the release commit.

## Completion Criteria

- `origin/main` and local `main` point to the merged v0.15.0 release commit.
- Tag `v0.15.0` points to that commit.
- GitHub Release `v0.15.0` is published successfully.
- Both plugin manifests and the dated changelog section report `0.15.0`.
- The local Copilot CLI installation reports Crucible `v0.15.0`.
- Temporary release/default worktrees and local branches are removed.
