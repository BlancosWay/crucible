# Crucible v0.15.0 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish Crucible v0.15.0 from the merged default-model change, synchronize the local repository, and update the locally installed Copilot CLI plugin.

**Architecture:** Cut release metadata on `release/v0.15.0`, validate it with the repository's deterministic release checks, and merge it through a pull request. Tag the exact merged `main` commit so the existing workflow publishes the GitHub Release, then fast-forward the primary checkout and refresh the locally registered marketplace plugin.

**Tech Stack:** Git, GitHub CLI, GitHub Actions, JSON manifests, Markdown changelog, Python 3.11+, Copilot CLI plugins

## Global Constraints

- Release version is exactly `0.15.0`; tag is exactly `v0.15.0`.
- Release date is exactly `2026-07-12`.
- The release contains the default pairing merged in PR #57: Claude Opus 4.8 max Builder and GPT-5.5 xhigh Critic.
- Keep `## [Unreleased]` at the top of `CHANGELOG.md` and move its current notes into `## [0.15.0] - 2026-07-12`.
- `.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json` must both report `0.15.0`.
- Do not change runtime code, schemas, CLI behavior, run logs, released changelog sections, or historical dated documents.
- Open and merge a release pull request before tagging; never tag an unmerged branch.
- Create a lightweight tag on the exact merged `main` commit; never move or force-update the tag.
- Let `.github/workflows/release.yml` publish the GitHub Release; do not create it manually.
- Update the local plugin only after the GitHub Release succeeds and local `main` is synchronized.
- Preserve the current release worktree until task and whole-branch reviews are complete.

- [ ] **Step 7: Wait for the tag-triggered Release workflow**

```bash
RUN_ID=""
TAG_SHA="$(git rev-list -n 1 v0.15.0)"
for attempt in $(seq 1 24); do
  RUN_ID="$(gh run list \
    --repo BlancosWay/crucible \
    --workflow Release \
    --commit "$TAG_SHA" \
    --event push \
    --limit 1 \
    --json databaseId \
    --jq '.[0].databaseId // empty')"
  if [ -n "$RUN_ID" ]; then
    break
  fi
  sleep 5
done
test -n "$RUN_ID"
gh run watch "$RUN_ID" --repo BlancosWay/crucible --exit-status
```

Expected: the Release workflow completes successfully. Do not continue after a failed conclusion.
